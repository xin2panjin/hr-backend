# 知识库文本分段与清洗预览改造方案

## 1. 目标

在“企业制度知识库”上传流程中增加类似 Dify 的“文本分段与清洗（含预览）”能力。管理员可在不写入数据库、文件存储、Milvus 或调用 Embedding 的前提下，查看原文经过清洗和结构化切片后的结果；确认配置后再上传并使用同一配置建立索引。

本次不建设通用低代码知识库平台，也不改变现有候选人检索链路。实现应保留为可复用于未来产品客服等知识库的通用能力。

## 2. 核心约束

- 预览为纯计算：不保存上传文件，不创建文档、切片或索引任务，不调用 Embedding、Milvus、LLM。
- 上传确认会把“清洗配置 + 切片配置”写入该索引任务的 `task_metadata`，任务执行时只能读取自身快照，不能读取后来被修改的前端默认值。
- 重建索引默认复用当前有效切片配置；管理员也可以在重建时显式传入一份新配置。
- 配置异常、文件类型/大小异常在预览和上传两条路径中采用相同的校验规则。
- 保留 Markdown 标题和 DOCX 标题层级，不能把结构化章节清洗掉。

## 3. 可调参数

### 3.1 清洗参数

| 参数 | 默认值 | 含义 |
| --- | --- | --- |
| `normalize_whitespace` | `true` | 合并连续空格、规范换行，保留段落与列表的基本结构 |
| `remove_urls_and_emails` | `false` | 替换 URL 和邮箱为占位文本，适合包含外部联系信息的文档 |
| `remove_blockquote_metadata` | `false` | 移除 Markdown 顶部连续引用行（常用于样例说明或元数据），避免干扰正式制度召回 |

### 3.2 切片参数

| 参数 | 默认值 | 含义 |
| --- | --- | --- |
| `max_characters` | `500` | 单个切片正文最大字符数，合法范围 50–2000 |
| `overlap_characters` | `80` | 相邻切片重叠字符数，必须小于最大长度 |

首版固定采用“章节/页码优先 + 句末标点切分”的结构化策略。自定义分隔符、语义切分模型和 OCR 清洗作为后续增强，不在本期引入。

## 4. 设计

```mermaid
flowchart LR
    File[选择本地文件] --> PreviewAPI[POST /knowledge/documents/preview]
    PreviewAPI --> Extract[文本提取]
    Extract --> Clean[纯文本清洗]
    Clean --> Chunk[结构化切片]
    Chunk --> Preview[返回统计与切片预览]

    Preview --> Confirm[确认上传]
    Confirm --> Register[保存原始文件 + 创建任务]
    Register --> TaskMeta[task_metadata 配置快照]
    TaskMeta --> IndexTask[异步索引任务]
    IndexTask --> Extract2[提取 + 同配置清洗 + 切片]
    Extract2 --> Embed[Embedding]
    Embed --> Milvus[Milvus]
```

### 4.1 后端模块职责

- `services/knowledge_text_processing.py`：通用配置模型、文本清洗器、结构化切片器；不依赖数据库、Web 框架或外部服务。
- `services/knowledge_preview_service.py`：预览文件的临时落盘、文本提取、调用清洗/切片，并在 finally 中删除临时文件。
- `services/knowledge_index_service.py`：从任务元数据解析配置快照，执行与预览完全相同的处理链路。
- `services/knowledge_document_service.py`：上传文件时校验并把配置传给索引任务；重建时继承最近成功任务的配置或接受新配置。
- `routers/knowledge_router.py`：受 `knowledge.document_manage` 保护的预览、上传和重建 API。
- `schemas/knowledge_schema.py`：请求/响应与配置 Schema。

### 4.2 API 契约

| 接口 | 用途 | 是否写入 |
| --- | --- | --- |
| `POST /knowledge/documents/preview` | 上传临时文件，返回清洗与切片预览 | 否 |
| `POST /knowledge/documents` | 上传并创建索引任务，接收 `processing_config` JSON | 是 |
| `POST /knowledge/documents/{id}/reindex` | 重建；可选接收 `processing_config` JSON | 是 |

预览响应包含：原始/清洗后块数、切片数、总字符数、每个切片的章节、页码、字符数、估算 token 数与正文。为防止一次响应过大，最多返回前 100 个切片；统计仍基于完整结果。

### 4.3 前端交互

知识库页面增加“文本分段与清洗”折叠区：

1. 选择文件后可调整清洗复选项、最大分段长度与重叠长度；
2. 点击“预览分段”调用预览 API；
3. 右侧/下方展示统计、切片列表和可展开的正文；
4. 修改任一参数或文件后使旧预览失效；
5. “上传并建立索引”携带当前配置；未预览时也可使用默认配置上传。

## 6. 验收标准

- 对同一文件、同一配置，预览返回的切片正文与实际索引前生成的正文一致。
- 预览调用后，数据库文档/任务/切片数量和 Milvus 数据均不变化。
- 上传任务的 `task_metadata.processing_config` 保存完整有效配置；重试不受后续前端改动影响。
- 没有 `knowledge.document_manage` 权限的用户不能预览、上传或重建。
- 修改参数或文件后，前端清晰提示需重新预览，不把旧预览误认为当前结果。
